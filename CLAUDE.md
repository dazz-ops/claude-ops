# claude-ops

Autonomous agent orchestration system. Schedules Claude Code CLI invocations across three roles (PM, Developer, Tech Lead) targeting repos with the godmode protocol installed.

## Quick Start

```bash
./scripts/install.sh          # Check deps, configure targets, generate crontab
./scripts/install.sh --check  # Check dependencies only
./scripts/dispatch.sh --role product-manager --target <name> --task "triage issues" --dry-run
./scripts/status.sh           # Dashboard
```

## Architecture

```
claude-ops/
├── config.template.json        # Template (committed) — install.sh generates config.json
├── config.json                 # Machine-specific (gitignored)
├── roles/                      # Agent personas (prompt + tool restrictions)
│   ├── product-manager.md      # Read-only: triage, enhance, explore, ideate
│   ├── developer.md            # Read-write: implement, self-review, PR
│   └── tech-lead.md            # Read-only: architecture, patterns, tech debt
├── jobs/                       # Cron job scripts (wrappers around dispatch.sh)
│   ├── pm-triage.sh            # Daily: categorize and prioritize issues
│   ├── pm-enhance.sh           # Daily: flesh out needs_refinement issues
│   ├── pm-explore.sh           # Weekly: explore codebase + ideate new features
│   ├── dev-implement.sh        # Daily: implement → self-review → fix → PR
│   └── tech-lead-review.sh     # Weekly: architecture review
├── schedules/
│   └── crontab                 # Generated by install.sh
├── scripts/
│   ├── install.sh              # Setup: deps, auth, config, crontab
│   ├── dispatch.sh             # Core dispatcher: loads role, invokes claude -p
│   ├── status.sh               # Dashboard
│   └── log-cleanup.sh          # Weekly cleanup
├── state/                      # Runtime (gitignored)
└── logs/                       # Run logs (gitignored)
```

## Schedule

```
08:00 Mon  PM Explore/Ideate  — discover gaps, brainstorm features, file issues
09:00      PM Triage           — categorize and prioritize new issues
10:00      PM Enhance          — flesh out needs_refinement → ready_for_dev
11:00      Developer           — implement → fresh-eyes review → fix → create PR
15:00 Fri  Tech Lead           — weekly architecture review
03:00 Sun  Log Cleanup
```

## Flow

```
PM files/enhances issues → [ready_for_dev] → Developer implements + self-reviews → [PR]
  → Human merges → Tech Lead reviews architecture (Friday)
```

The Developer runs `/fresh-eyes-review` on its own code and fixes all findings before creating the PR. No separate QA step needed — the PR arrives already reviewed.

Human merges PRs after inspecting them. No agent can merge.

## Safety

- Three roles, clear boundaries: PM reads + files issues, Developer writes code, Tech Lead reads + advises
- Developer self-reviews before PR creation (fresh-eyes review)
- Read-only roles skip target lock (safe to overlap)
- Read-write roles verify clean working tree before dispatch
- No agent can merge PRs, push to main, or force push
- Daily invocation cap (default 20/day)
- `--dangerously-skip-permissions` required for headless mode — tool access enforced via `--allowedTools`

## Installation

```bash
git clone <repo-url> && cd claude-ops
./scripts/install.sh
```

Requires: bash, jq, git, claude (Claude Code CLI), gh (GitHub CLI authenticated with repo scope)
