# claude-ops

Autonomous agent orchestration system. Schedules and coordinates Claude Code CLI invocations across roles (PM, Developer, QA, Tech Lead) targeting repos that have the godmode protocol installed.

## Quick Start

```bash
# Install and configure
./scripts/install.sh

# Check dependencies only
./scripts/install.sh --check

# Test a dry run
./scripts/dispatch.sh --role product-manager --target <name> --task "triage issues" --dry-run

# Check status
./scripts/status.sh
```

## Architecture

```
claude-ops/
├── config.template.json        # Template (committed) — install.sh generates config.json
├── config.json                 # Machine-specific config (gitignored, generated)
├── roles/                      # Agent persona definitions (prompt + tool restrictions)
│   ├── product-manager.md      # Read-only: triage, enhance issues, explore
│   ├── developer.md            # Read-write: implement, test, PR, fix
│   ├── qa-engineer.md          # Read-only: review PRs, run tests, report
│   └── tech-lead.md            # Read-only: architecture, patterns, tech debt
├── jobs/                       # Cron job scripts (thin wrappers around dispatch.sh)
│   ├── pm-triage.sh            # Daily 9am: categorize and prioritize issues
│   ├── pm-enhance.sh           # Daily 10am: flesh out needs_refinement issues
│   ├── pm-explore.sh           # Monday 8am: discover improvement opportunities
│   ├── dev-implement.sh        # Daily 11am: implement next ready_for_dev issue
│   ├── dev-fix-pr.sh           # Daily 4pm: fix PRs with QA findings
│   ├── qa-review-prs.sh        # Daily 2pm: review open PRs
│   └── tech-lead-review.sh     # Friday 3pm: architecture review
├── schedules/
│   └── crontab                 # Generated by install.sh with correct paths
├── scripts/
│   ├── install.sh              # Setup: deps, auth, config, crontab generation
│   ├── dispatch.sh             # Core dispatcher: loads role, invokes claude -p
│   ├── status.sh               # Dashboard: budget, locks, recent runs
│   └── log-cleanup.sh          # Weekly cleanup of old logs
├── state/                      # Runtime state (gitignored)
│   ├── budget-YYYY-MM-DD.json
│   ├── invocations.jsonl
│   └── locks/
└── logs/                       # Run logs (gitignored)
```

## Daily Schedule

```
08:00 Mon  PM Explore     — discover improvement opportunities (weekly)
09:00      PM Triage      — categorize and prioritize new issues
10:00      PM Enhance     — flesh out needs_refinement → ready_for_dev
11:00      Dev Implement  — pick and implement next ready_for_dev issue
14:00      QA Review      — review open PRs, post findings
15:00 Fri  Tech Lead      — weekly architecture review
16:00      Dev Fix        — fix PRs flagged by QA
03:00 Sun  Log Cleanup    — remove old log files
```

## Coordination Model

Agents coordinate through GitHub issues and PRs — no direct communication.

```
PM creates/enhances issues → [GitHub: ready_for_dev]
  → Developer implements, creates PR → [GitHub: open PR]
    → QA reviews PR, posts findings (<!-- claude-ops:qa-review -->)
      → Developer fixes findings (<!-- claude-ops:dev-fix -->)
        → QA re-reviews on next run
          → Human merges
            → Tech Lead reviews architecture (Friday)
```

## Safety

- Read-only roles (PM, QA, Tech Lead) cannot modify code — enforced by `--allowedTools`
- Read-only roles run without target locks (can overlap safely)
- Read-write roles (Developer) acquire exclusive target lock + verify clean working tree
- Developer cannot merge PRs, push to main, or force push
- Daily invocation cap (configurable, default 20/day)
- All output logged for audit
- Stale lock detection with PID validation
- `--dangerously-skip-permissions` is required for headless `claude -p` — tool restrictions enforced via `--allowedTools`

## Installation

Requires: bash, jq, git, claude (Claude Code CLI), gh (GitHub CLI)

```bash
git clone <repo-url> && cd claude-ops
./scripts/install.sh
```

The installer checks all dependencies, verifies GitHub/Claude authentication, lets you configure target repos interactively, and generates machine-specific config.json and crontab.
